Crawlr is an AI-powered, self-healing web scraper for e-commerce price & stock monitoring. When a site changes its layout, Crawlr repairs its own selectors — so your monitoring never silently dies.

Get started Read the docs

Most scrapers break the moment a site changes.

Crawlr uses an LLM (or a free offline heuristic) to generate and repair CSS selectors — then extracts pages deterministically with those cached selectors. The model runs once per site, not on every page: fast, cheap, and resilient. 

$ crawlr watch "https://store.example/product/123" --target 25
Watching site #1 (trigger=price_below, target $25, every 60m)

How it works

The self-healing loop that keeps monitoring alive.

1

Fetch & simplify

Crawlr fetches the page (static or JS-rendered) and strips it to a compact outline.

2

Generate selectors

An LLM or the offline heuristic produces reusable CSS selectors, cached per site.

3

Extract deterministically

Every subsequent run reuses cached selectors — fast, free, and consistent.

4

Self-heal on breakage

If selectors stop matching after a redesign, Crawlr regenerates them and continues.

5

Detect changes

Snapshots are diffed against the previous run to catch price and stock movements.

6

Alert you

Matching changes fire alerts through your chosen triggers and channels.

Quickstart

Up and running in under a minute — no API key required.

# Install
git clone https://github.com/ardfaiyaz/crawlr.git
cd crawlr
pip install -e ".[dev]"

# Watch a product's price and stock
crawlr watch "https://store.example/product/123" --target 25

# Check your watchlist
crawlr watchlist

# Keep monitoring in the background
crawlr monitor --daemon
