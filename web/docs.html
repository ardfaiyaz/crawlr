What is Crawlr?

Crawlr is a command-line tool that watches product pages and tells you when the price or stock changes. Point it at a link, pick when you want to be alerted, and it keeps checking for you.

Under the hood it's a self-healing scraper: it uses AI (or a free offline method) only to work out the right CSS selectors once, then reuses them — fast and free. If a site changes its layout and extraction breaks, Crawlr notices and fixes the selectors automatically, so your monitoring doesn't silently die.

1. Install

You need Python 3.10 or newer. Install with pipx (recommended) or pip:

bash
pipx install crawlr        # or: pip install crawlr

Optional add-ons, only if you need them:

bash
pip install "crawlr[js]" && playwright install chromium   # JavaScript-heavy sites
pip install "crawlr[postgres]"                            # use Postgres instead of SQLite

Check everything works:

bash
crawlr doctor

2. Your first watch

Three steps: watch a product, see your list, then let it keep checking.

bash
# Step 1 — start watching (alert me at/below $25). Schema is auto-detected.
crawlr watch "https://store.example/product/123" --target 25

# Step 2 — see current price, movement, stock, and status
crawlr watchlist

# Step 3 — keep checking in the background (or use cron)
crawlr monitor --daemon

New to scraping? Try it first on a scrape-friendly demo site to see it work end-to-end:

bash
crawlr watch "https://books.toscrape.com/catalogue/a-light-in-the-attic_1000/index.html" --target 60
crawlr watchlist
crawlr monitor

How it works

Every scrape flows through a layered pipeline where each stage cross-checks the next:

  1. Fetch — static HTTP with retries; escalates to a headless browser (or a fetch provider) for hard pages. Honors robots.txt and rate limits.
  2. Structured data — reads schema.org JSON-LD, microdata, and OpenGraph — the most reliable, redesign-proof source.
  3. Selectors — cached CSS selectors extract deterministically and self-heal when they break.
  4. Fallbacks — if those miss, page-level heuristics still find the title/price/stock (even a currency-tagged number in the text).
  5. Consensus & validate — sources are merged, disagreements resolved, types checked, and absurd values (e.g. a 100× jump) rejected.
  6. Alert — only real, meaningful changes reach you, via the trigger you chose.

Watch options

Add these flags to crawlr watch <url> to control what it does:

OptionWhat it does
--target 500Ping me when the price drops to 500 or below (also sets the trigger to price_below)
--restockPing me when the item goes from out of stock → in stock
--trigger <name>Choose exactly when to be alerted (see triggers below)
--interval 60Check this page every 60 minutes (default: 60)
--schema <name>Force the page type instead of auto-detecting: product, product_list, jobs, real_estate, news
bash
# "Tell me when this drops to 25 or lower, checking every 30 minutes"
crawlr watch "https://store/p/1" --target 25 --interval 30

Alert triggers

Pick one per watch with --trigger (or in the dashboard). In plain English:

TriggerAlerts you when…
any_changeanything you watch changes (price, stock, title…)
price_dropthe price goes down by any amount
price_belowthe price is at or below your --target
price_abovethe price is at or above your --target
back_in_stockthe item becomes available again
out_of_stockthe item sells out

Recipes

Copy-paste starting points for common goals:

bash
# Track a competitor's price, alert on any drop, check hourly
crawlr watch "https://store.com/p/123" --trigger price_drop --interval 60

# Get pinged when something is back in stock
crawlr watch "https://store.com/p/123" --restock

# Compare the same product across three shops
crawlr compare "https://a.com/p" "https://b.com/p" "https://c.com/p"

# Monitor a job board (schema auto-detected)
crawlr watch "https://boards.example/jobs" --interval 720

# Free phone alerts via ntfy.sh, then run in the background
export CRAWLR_ALERT_NTFY="https://ntfy.sh/my-crawlr-alerts"
crawlr monitor --daemon

# Pause / resume / remove a watch (ids come from `crawlr watchlist`)
crawlr pause 3 && crawlr resume 3 && crawlr unwatch 3

# Pipe machine-readable output into other tools
crawlr watchlist --json | jq '.[] | {title, price}'

Canvas — shop a product across many stores

Know what you want but not where to buy it (say a Wooting 60HE)? crawlr canvas searches several retailers at once, grabs the best-matching result and price from each, converts everything to one currency, and ranks them so you can comparison-shop in one command.

bash
crawlr canvas "Wooting 60HE"                          # global stores
crawlr canvas "MAD60 HE" --country ph                 # Lazada PH, Shopee PH, Zalora PH…
crawlr canvas "Wooting 60HE" --to PHP                 # infers --country ph from the currency
crawlr canvas "Wooting 60HE" --retailers amazon,ebay --to USD

Canvas is location-aware. Pass --country (an ISO code like ph, sg, us) — or set CRAWLR_COUNTRY, or just use a local currency with --to — and Crawlr searches that country's stores instead of the global set:

Country--countryLocal stores it searches
PhilippinesphLazada PH, Shopee PH, Zalora PH, AliExpress
SingaporesgLazada SG, Shopee SG, Amazon SG, AliExpress
MalaysiamyLazada MY, Shopee MY, AliExpress
IndonesiaidLazada ID, Shopee ID, Tokopedia, AliExpress
ThailandthLazada TH, Shopee TH, AliExpress
VietnamvnLazada VN, Shopee VN, Tiki, AliExpress
United StatesusAmazon, eBay, Walmart, Newegg, Best Buy
United KingdomgbAmazon UK, eBay UK, Currys, AliExpress
IndiainAmazon IN, Flipkart, AliExpress

JapanjpAmazon JP, AliExpress
CanadacaAmazon CA, eBay CA, Newegg CA

With no country and no currency hint, Crawlr uses the global stores. Add your own shops any time with a YAML file (CRAWLR_CANVAS_RETAILERS).

Important: local marketplaces like Lazada and Shopee block bots (just like Amazon). For real results from them, set up a fetch provider first — otherwise those rows show "blocked or unreachable".

Configuration — how settings work

Crawlr is configured with environment variables. You can set them three ways:

WhereHow
macOS / Linuxexport CRAWLR_LLM_PROVIDER=openai
Windows PowerShell$env:CRAWLR_LLM_PROVIDER="openai"
A .env fileAdd a line CRAWLR_LLM_PROVIDER=openai in your project folder

The .env file is the easiest for permanent settings — Crawlr loads it automatically. Below are the settings grouped by what you're trying to do; there's a full reference table at the end.

AI mode (LLM) — optional, improves accuracy

Crawlr runs free and offline by default. Adding an OpenAI or Anthropic key only helps on unusual layouts, and the model runs once per site so cost stays tiny.

OpenAI — get a key:

  1. Go to platform.openai.com and sign in.
  2. Open the account menu (top-right) → API keys → Create new secret key.
  3. Copy the key now — it's only shown once (starts with sk-).
  4. Add a payment method under Settings → Billing (API usage needs credit).
  5. Set it:
bash
export CRAWLR_LLM_PROVIDER="openai"
export CRAWLR_LLM_API_KEY="sk-..."
export CRAWLR_LLM_MAX_CALLS="2"   # optional: cap calls per scrape (cost guard)

Anthropic — get a key:

  1. Go to console.anthropic.com and sign in.
  2. Open API keys → Create Key → copy it (starts with sk-ant-).
  3. Set it:
bash
export CRAWLR_LLM_PROVIDER="anthropic"
export CRAWLR_LLM_API_KEY="sk-ant-..."

To go back to free offline mode: export CRAWLR_LLM_PROVIDER="none".

Alerts & notifications — step by step

Pick a channel below, follow the steps to get its URL/token, set the variable, then run crawlr test-alert to confirm the message arrives. You can enable several at once.

Discord

  1. Open Discord and go to the server you manage where you want alerts.
  2. Click the server name → Server Settings.
  3. Go to Integrations → Webhooks → New Webhook.
  4. Choose the channel it should post in, (optionally) rename it, then click Copy Webhook URL.
  5. Set it and test:
bash
export CRAWLR_ALERT_DISCORD="https://discord.com/api/webhooks/…"
crawlr test-alert

Slack

  1. Go to api.slack.com/apps → Create New App → From scratch; pick your workspace.
  2. Open Incoming Webhooks and switch it On.
  3. Click Add New Webhook to Workspace, choose a channel, and Allow.
  4. Copy the webhook URL, then set it:
bash
export CRAWLR_ALERT_SLACK="https://hooks.slack.com/services/…"

Telegram

  1. In Telegram, message @BotFather, send /newbot, and follow the prompts. Copy the bot token it gives you.
  2. Open a chat with your new bot and send it /start (a bot can't message you until you do).
  3. Message @userinfobot — it replies with your numeric chat id.
  4. Set both:
bash
export CRAWLR_ALERT_TELEGRAM_TOKEN="123456:ABC-your-token"
export CRAWLR_ALERT_TELEGRAM_CHAT_ID="123456789"

ntfy — free phone push, no account

  1. Choose a hard-to-guess topic name, e.g. my-crawlr-a7f3. Your URL is https://ntfy.sh/my-crawlr-a7f3.
  2. Install the ntfy app (iOS/Android) — or open the URL in a browser — and subscribe to that topic.
  3. Set it:
bash
export CRAWLR_ALERT_NTFY="https://ntfy.sh/my-crawlr-a7f3"

Microsoft Teams

  1. In Teams, hover the channel → ⋯ → Workflows (older tenants: Connectors → Incoming Webhook).
  2. Pick the "post to a channel when a webhook request is received" template and create it.
  3. Copy the generated URL and set it:
bash
export CRAWLR_ALERT_TEAMS="https://…webhook…url…"

Email (SMTP)

  1. Get your provider's SMTP details. For Gmail: enable 2-step verification, then create an App Password (Google Account → Security → App passwords) and use that as the password.
  2. Set the recipient and server:
bash
export CRAWLR_ALERT_EMAIL_TO="you@example.com"
export CRAWLR_SMTP_HOST="smtp.gmail.com"
export CRAWLR_SMTP_PORT="587"
export CRAWLR_SMTP_USER="you@gmail.com"
export CRAWLR_SMTP_PASSWORD="your-app-password"

Any generic webhook

  1. Use any endpoint that accepts an HTTP POST with JSON. To try it out, grab a free URL from webhook.site.
  2. Set it (optionally sign the payloads so your receiver can verify them):
bash
export CRAWLR_ALERT_WEBHOOK="https://webhook.site/your-id"
export CRAWLR_WEBHOOK_SECRET="any-shared-secret"   # optional; adds X-Crawlr-Signature

Fine-tuning: add CRAWLR_ALERT_THROTTLE_MINUTES=60 so the same alert can't repeat within an hour. Prefer a daily summary over per-change pings? Use crawlr digest --send or run crawlr monitor --daemon --digest 24.

Currency conversion

crawlr compare and crawlr canvas can compare prices across currencies. By default they convert using a built-in offline rate table into USD.

bash
export CRAWLR_FX_BASE="PHP"          # compare everything in pesos
export CRAWLR_FX_LIVE="true"         # fetch live rates (cached), else use pinned table
export CRAWLR_FX_RATES="EUR=0.92,GBP=0.79"  # pin specific rates (units per USD)

crawlr fx --amount 100 --from EUR --to USD   # quick one-off conversion

Fetch providers — for big marketplaces

Amazon, Lazada, Shopee and friends actively block bots. Instead of fighting that, route fetches through a rendering/unblocking service (they run a real browser + rotate proxies and hand back clean HTML). Crawlr then extracts as usual — no code changes.

bash
export CRAWLR_FETCH_PROVIDER="scraperapi"   # or scrapingbee | zyte | custom
export CRAWLR_FETCH_PROVIDER_KEY="your-key"
crawlr doctor                              # confirms the provider is set

Getting a key (ScraperAPI shown; ScrapingBee & Zyte are the same idea):

  1. Sign up at scraperapi.com (there's a free trial tier).
  2. Open your dashboard and copy the API key shown there.
  3. Set the two variables above (CRAWLR_FETCH_PROVIDER="scraperapi" and your key), then run crawlr doctor.
  4. Watch a marketplace URL as normal — Crawlr now routes that fetch through the provider automatically.

Most providers offer a free tier to start. Any other "URL in → HTML out" service works via CRAWLR_FETCH_PROVIDER=custom plus CRAWLR_FETCH_PROVIDER_ENDPOINT, _URL_PARAM, _KEY_PARAM/_KEY_HEADER, _EXTRA, and _RESPONSE/_HTML_PATH. Even more reliable and fully legal: a marketplace's official API.

Anti-bot & robots

bash
export CRAWLR_PROXIES="http://user:pass@host:port,http://host2:port"  # rotate proxies
export CRAWLR_RESPECT_ROBOTS="false"  # only for sites you're allowed to scrape

Crawlr honors robots.txt by default. Turning it off is your responsibility — see Legal & ethics.

Storage & Postgres

Crawlr uses a local SQLite database by default (zero setup). For scale or multiple workers, point it at Postgres — the same code runs on both.

bash
export CRAWLR_DATABASE_URL="postgresql://user:pass@localhost:5432/crawlr"
export CRAWLR_DATA_DIR="./.crawlr"   # where the DB, cache & snapshots live

Easiest: a free hosted Postgres (Neon shown; Supabase/Railway are the same idea):

  1. Create a free project at neon.tech (or supabase.com / railway.app).
  2. Copy the connection string — it looks like postgresql://user:pass@host:5432/dbname.
  3. Install the Postgres extra and set the URL:
bash
pip install "crawlr[postgres]"
export CRAWLR_DATABASE_URL="postgresql://user:pass@host:5432/dbname"
crawlr doctor   # verifies the connection

Custom schemas (no code)

A schema tells Crawlr what to pull off a page. It ships with product and product_list, plus presets for jobs, real_estate, and news. You can add your own by writing a small YAML file — no Python needed.

Step 1 — write the schema. Each field has a name, a plain description (the hint Crawlr uses to find it), and a type:

jobs.yaml
name: jobs
item_selector: ".job-card"     # repeats once per result on a list page
fields:
  - name: title
    description: the job title
    type: text
    required: true             # extraction "fails" if this is missing
  - name: salary
    description: annual salary
    type: number
PieceMeaning
nameThe schema's name — you pass it to --schema.
item_selectorCSS selector for one repeating item (list/search pages). Omit it for single-item pages.
descriptionNatural-language hint used to locate the field the first time.
typetext, number, price, url, image, or bool.
requiredtrue means a run counts as "broken" (and self-heals) if this field is empty.

Step 2 — put it where Crawlr looks (your schema directory, by default ./.crawlr/schemas/) and validate it:

bash
mkdir -p .crawlr/schemas && mv jobs.yaml .crawlr/schemas/
crawlr validate-schema .crawlr/schemas/jobs.yaml   # checks it's well-formed
crawlr schemas                                     # confirm it's listed

Step 3 — use it:

bash
crawlr scrape "https://boards.example/jobs" --schema jobs

Rules template

Want richer "if this, do that" logic across many situations? Run crawlr init to create crawlr.rules.yaml. When it exists, it overrides per-watch triggers.

crawlr.rules.yaml
default_action: ignore
rules:
  - when: price_drops_below
    amount: 25
    action: alert
  - when: back_in_stock
    action: alert
  - when: price_increases
    action: ignore

Hosted API

Run the JSON API and drive Crawlr from any language. Set CRAWLR_API_KEY to require an X-API-Key (or Authorization: Bearer) header.

bash
CRAWLR_API_KEY=secret crawlr serve --host 0.0.0.0 --port 8000

# endpoints
POST /api/scrape     {"url": "...", "schema_name": "product"}
POST /api/watch      {"url": "...", "trigger": "price_below", "target_price": 25}
GET  /api/watchlist  |  /api/stats  |  /api/digest?hours=24  |  /api/changes

Deploy

Crawlr is a CLI — "deploy" just means running the checker on a timer:

  • Cron — schedule crawlr monitor however often you like.
  • Daemon — crawlr monitor --daemon on any always-on machine.
  • Docker — docker compose up -d runs the dashboard + a background monitor.

Legal & ethics

Crawlr honors robots.txt by default and rate-limits politely. You are responsible for how you use it. Respect each site's Terms of Service and applicable law (e.g. the CFAA and GDPR/CCPA when personal data is involved). Point it at sites you're permitted to monitor — your own listings, price research, or public data — and don't use it to overload servers, bypass logins/paywalls, or collect personal data. For big marketplaces, prefer official APIs where available.
