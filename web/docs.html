Install

Crawlr is a Python command-line tool (Python 3.10+).

git clone https://github.com/ardfaiyaz/crawlr.git
cd crawlr
python -m venv .venv
# macOS / Linux
source .venv/bin/activate
# Windows PowerShell
.venv\Scripts\Activate.ps1

pip install -e ".[dev]"

Optional extras:

pip install -e ".[js]" && playwright install chromium   # JS-heavy sites
pip install -e ".[postgres]"                            # Postgres backend
Crawlr runs fully offline with zero API keys using its heuristic selector engine. An LLM key is optional and only improves accuracy on unusual layouts.

Quickstart

crawlr watch "https://store.example/product/123" --target 25
crawlr watchlist
crawlr monitor --daemon

Watch & watchlist

The simplest way to monitor a competitor product:

crawlr watch "https://store.example/p/1"              # track price + stock
crawlr watch "https://store.example/p/1" --target 25  # alert at/below $25
crawlr watch "https://store.example/p/1" --restock    # alert when back in stock
crawlr watch "https://store.example/p/1" --interval 30 # check every 30 min

crawlr watchlist shows current price, previous price, % change, stock, target, and status.

Alert triggers

Pick when a watch should alert (CLI --trigger or the dashboard dropdown):

TriggerAlerts when
any_changeany watched field changes
price_dropthe price goes down
price_belowprice is at/below your target
price_aboveprice is at/above your target
back_in_stockthe item becomes available
out_of_stockthe item sells out

Rules template

For richer logic across many situations, generate an editable rules file:

crawlr init   # writes crawlr.rules.yaml
default_action: ignore
rules:
  - when: price_drops_below
    amount: 25
    action: alert
  - when: back_in_stock
    action: alert
  - when: price_increases
    action: ignore

When crawlr.rules.yaml exists it takes precedence over per-watch triggers — one place to describe exactly what should happen in each circumstance.

Custom schemas (no code)

Add new verticals by dropping a YAML file into your schema directory:

name: jobs
item_selector: ".job-card"
fields:
  - name: title
    description: the job title
    type: text
    required: true
  - name: salary
    description: annual salary
    type: number
crawlr validate-schema ./jobs.yaml
crawlr scrape "https://jobs.example" --schema jobs

Alerts

Configure any combination of sinks via environment variables:

export CRAWLR_ALERT_WEBHOOK="https://webhook.site/your-id"
export CRAWLR_ALERT_SLACK="https://hooks.slack.com/services/..."
export CRAWLR_ALERT_EMAIL_TO="you@example.com"
export CRAWLR_SMTP_HOST="smtp.example.com"

AI mode (optional)

Enable LLM selector generation for tricky sites. Usage is tiny — the model runs once per site, then selectors are cached.

export CRAWLR_LLM_PROVIDER="openai"   # or anthropic
export CRAWLR_LLM_API_KEY="sk-..."
export CRAWLR_LLM_MODEL="gpt-4o-mini"

Deploy

Crawlr is a CLI, so "deploy" means running the scheduler somewhere it can check on a timer. Options:

  • Cron / Task Scheduler: run crawlr monitor on a schedule.
  • Daemon: run crawlr monitor --daemon on any always-on machine.
  • Docker: docker compose up starts the scheduler with Postgres.
This website is a static showcase and deploys free on Vercel. The product itself is the command-line tool — install it locally or on a server.
